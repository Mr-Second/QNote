# QNote v0.3.0 Release Notes

## 🔧 性能改进

### 内存占用优化

本版本针对 QNote 运行时内存占用进行了两项优化，显著降低日常使用和后台驻留时的内存开销。

#### 1. 图形后端切换为 D3D11（可见态 -70MB）

- 之前强制使用 OpenGL 图形后端（项目初始模板代码），在核显机器上会额外产生 ~70MB 私有内存开销
- 改为 Qt 6 在 Windows 上的默认 D3D11 后端，驱动层原生支持，GPU 内存由驱动管理
- 实测 PrivateMemory 从 320MB 降至 250MB（-70MB，-22%）

#### 2. 窗口隐藏时释放 GPU 资源（托盘驻留态 -80MB）

- 主窗口设置 `setPersistentSceneGraph(false)` + `setPersistentGraphics(false)`
- 最小化到托盘 / 关闭到托盘时，Qt 自动释放 scene graph 和图形上下文
- 显现时自动重建，配合 opacity 淡入掩盖重建过程，无黑块/白屏
- 最小化拦截：最小化操作转为 `hide()`，确保触发资源释放
- 实测 PrivateMemory 从 250MB 降至 170MB（-80MB，-32%）

### 综合效果

| 场景 | v0.2.0 (OpenGL) | v0.3.0 可见态 (D3D11) | v0.3.0 托盘态 (D3D11+释放) |
|------|-----------------|----------------------|---------------------------|
| PrivateMemory | ~320 MB | ~250 MB | ~170 MB |

## 📊 技术背景

前序调研（v0.2.0 后）发现 QNote 日常使用态 ~320MB，远超 Qt 6 QML 正常基线。经 VMMap 堆分析 + 代码插桩诊断 + 排除实验，确认：
- 非内存泄漏（启动即 250MB，非渐进增长）
- 非应用层浪费（移除 133 个 QML item 仅省 1.1MB）
- 主因是 OpenGL 图形后端开销 + 窗口可见时 GPU 资源常驻
- Qt 6 QML 在 Windows 上 ~250MB（D3D11 后）是框架基线（社区共识）

本版本在不重写应用的前提下，通过图形后端 + 资源释放策略获得最大收益。
